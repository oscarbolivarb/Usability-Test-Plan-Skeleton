# Usability-Test-Plan-Skeleton

Test Title 
Usability Test Plan Skeleton for a Medical Appointment Scheduling System.
System Overview
The selected system is a Medical Appointment Scheduling System. This system allows patients to create accounts, schedule appointments, view upcoming appointments and cancel or reschedule visits when needed. The purpose of the system is to make it easier for the patients to manage their healthcare appointments online instead of calling the clinic. 
The intended users would be patients with different levels of computer experience. The system helps improve convenience making it easier for users to manage their appointment much quicker. 
Purpose and Research Questions
Purpose
The main purpose of this usability test is to evaluate how easy it is for users to complete common tasks with the Medical Appointment Scheduling System. The results from this study will help identify usability problems and determine whether any improvements are needed before the system is fully implemented.
Research Questions 
1.	Can users successfully schedule an appointment without any assistance? 
2.	Where do users experience any confusions while using the system?
3.	Can users easily find and manage their existing appointments?
4.	Are users satisfied with the overall experience of the system?
Methodology and Participants 
Methodology 
This study will use an assessment usability test because the system has already been developed and the goal is to evaluate how effectively users can perform important tasks.
The test will be moderated and conducted remotely. A moderator will observe each participant during the session and record any usability issue that can occur. 
Participant Characteristics
The target participants include adults between the ages of 18 and 65 who have basic experience using computers or smartphone devices. The participants should have previous experience scheduling healthcare appointments.
Inclusion Criteria
•	Adults who regularly use websites or mobile applications. 
•	Users who are able to read and understand English.
Exclusion Criteria 
•	The employees who are involved in developing the system. 
•	Participants who have already taken part in this study.
Task List and Success Criteria 
Task 1: Schedule a New Appointment
Participants will schedule an appointment with a healthcare provider for a future date, 
Success Criteria: the participant reaches the appointment confirmation page without any help from the moderator.
Task 2: View Upcoming Appointments
The participants will locate and review their scheduled appointments.
Success Criteria: the participant successfully finds the correct appointment information.
Task 3: Reschedule an Existing Appointment
The participants will change the date and the time of an existing appointment. 
Success Criteria: the participant successfully saves the updated appointment information. 
Task 4: Cancel an Appointment
The participants will cancel an existing appointment.
Success Criteria: The participants will receive a confirmation showing that the appointment was canceled. 
Performance Measures and Benchmarks 
Quantitative Measures 
•	Task completion rate.
•	Time required to complete each task.
•	The number of errors made by participants.
Qualitative Measure 
•	  User satisfaction ratings.
•	Participant comments and feedback.
•	Severity of usability issues identified during testing.
Benchmarks 
Task 1 Benchmark
The participants should be able to complete the appointment scheduling task in less than two minutes with a completion rate of at least 90%.
Task 3 Benchmark
The participants should be able to reschedule an appointment in less than one minute and make sure there is no more than one error.
Logistics and Environment
The usability study will be conducted remotely using a computer with internet access and a screen recording software will be used to capture user interaction during the sessions. 
Each session will last around 30 minutes and the moderator’s responsibilities will be:
•	To explain the purpose of the study.
•	Provide instructions regarding the tasks.
•	Remain neutral and avoid giving hints.
•	Recording observations and participants feedback.
All participants will start with the same conditions and use the same version of the system
Data Collection and Risks
Data Collection 
Data will be collected through screen recording, observation notes, task completion times and participant feedback. All information gathered during the study will be stored electronically and reviewed after the test is completed. 
Risks and Mitigation 
Risk 1: Technical Problems
Issues with the internet connection or failure in the recordings could interrupt the testing session
Mitigation: all software and equipment will be tested before each session and backup recording methods will be available.
Risk 2: Recruitment Challenges
It may be difficult to find enough participants who match the target user profile.
Mitigation: the participants will be recruited early and additional backup participants will be identified if they are needed.
